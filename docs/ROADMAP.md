# DeepSpider 发展路线图

> 最后更新：2026-03-05

## 项目定位

**核心定义**：
DeepSpider 是一个 **AI 驱动的爬虫逆向分析工具**，帮助开发者快速理解网站的反爬机制并生成可维护的爬虫代码。

**价值主张**：
把 3 天的逆向分析工作压缩到 10 分钟。

**目标用户**：
- 主要：爬虫工程师（熟悉 Python/JavaScript）
- 次要：数据分析师（需要数据但不会逆向）

**核心能力**：
- AI 理解混淆代码
- 真实浏览器 + CDP 完整分析
- 端到端方案（分析 → 生成 → 验证）

**不做什么**：
- ❌ 通用爬虫框架（Scrapy 已足够）
- ❌ 可视化爬虫（八爪鱼已足够）
- ❌ 数据清洗（Pandas 已足够）
- ✅ 只聚焦逆向分析

## 架构演进

### 当前架构（v0.5）

```
主 Agent
├── reverse-agent（逆向分析）
├── crawler-agent（爬虫生成）
├── captcha-agent（验证码）
└── anti-detect-agent（反检测）

工具：90+
依赖：40+
代码：~8000 行
```

### 目标架构（v1.0）

```
DeepSpider Core（逆向分析引擎）
├── 数据层：浏览器 + CDP + Hook
├── AI 层：智能路由 + 多模型
└── 输出层：代码生成 + 验证

工具：30-40
依赖：< 20
代码：< 5000 行
```

**简化原则**：
- 删除非核心子代理（captcha、anti-detect、crawler）
- 精简工具集（保留核心 30-40 个）
- 移除过度设计（webcrack、isolated-vm）

## 技术路线

### 成本优化策略

```
智能路由：
├── Level 0：规则引擎（免费，< 100ms）
│   └── 适用：简单场景（明文 API、标准加密）
├── Level 1：本地模型（$0.01，< 2s）
│   └── 适用：常见场景（常规混淆、已知模式）
├── Level 2：云端辅助（$0.5，< 10s）
│   └── 适用：复杂场景（深度混淆、自定义加密）
└── Level 3：云端自主（$2，< 60s）
    └── 适用：极端场景（对抗性反爬、动态策略）
```

**目标成本分布**：
- 70% 请求：规则引擎（免费）
- 20% 请求：本地模型（$0.01）
- 10% 请求：云端 AI（$0.5-2）
- 平均成本：< $0.3/次
