> 模板用途：追踪单个目标请求从发现到验证的完整生命周期。
> 每个加密参数独立填写一份标签块。

---

## 生命周期状态

```
draft → captured → traced → evidence-complete → recovering → validated → handoff-ready
```

**当前状态：** `draft` <!-- 更新为当前状态 -->

| 状态 | 含义 |
|------|------|
| draft | 已识别请求，尚未完整抓包 |
| captured | 请求已完整捕获，Headers/Params 已记录 |
| traced | 已通过断点/Call Stack 定位到加密源码 |
| evidence-complete | 算法逻辑已确认，证据充分 |
| recovering | Python 实现编写中 |
| validated | 多样本验证通过 |
| handoff-ready | 可交付爬虫使用 |
