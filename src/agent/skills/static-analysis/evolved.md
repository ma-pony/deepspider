### [2026-03-03] 混淆代码中的字符串常量提取
**场景**: 在分析安徽省信用信息公示平台的 sign 加密时，发现常量 _tt 直接定义在代码中，但 _gg 和 _yy 通过混淆函数 _0x2e4474() 动态获取。通过静态分析代码结构，成功提取了 _tt 的完整值，并理解了密钥处理逻辑（长度检查、截取后128字符、分割为两部分）。
**经验**: 对于混淆代码，优先搜索直接定义的字符串常量（如 _tt='...'），这些往往是关键密钥。对于通过函数调用的常量（如 _gg=_0x2e4474(0x15d)），可以尝试通过搜索函数名和参数来定位。

### [2026-03-03] 多层哈希嵌套模式识别
**场景**: 在分析安徽省信用信息公示平台的 radms 函数时，发现其使用了 SHA1 → Base64 → SHA1 的三层嵌套哈希模式。用户代码只进行了一次 SHA1，而实际代码进行了三次（两次 SHA1 中间夹 Base64）。通过逐行分析混淆代码的 CryptoJS 调用链，识别出了这个关键差异。
**经验**: 当遇到哈希计算结果不匹配时，检查是否存在多层嵌套哈希（如 SHA1(SHA1(Base64(SHA1(data))))）。混淆代码中的 CryptoJS 调用链往往是关键线索。

### [2026-03-02] 混淆JS中硬编码密钥定位技巧
**场景**: 在分析 credit.ah.gov.cn 的 sign 参数加密时，发现密钥通过混淆代码中的变量赋值。通过搜索 `_aa=`, `_bb=` 等模式，成功定位到硬编码的 SM2 公钥和常量。
**经验**: 混淆代码中的硬编码密钥通常以简单变量赋值形式存在，搜索 `varName=` 或 `constName=` 模式比搜索密钥内容更有效。
